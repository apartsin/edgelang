User Manual

Set Up EdgeLang and Keep It Moving

This manual covers installation, configuration, everyday use, and the quickest way to diagnose why cues are or are not showing up on the page.

Install

  1. Open Chrome and go to chrome://extensions.
  2. Enable Developer mode.
  3. Click Load unpacked.
  4. Select the repository src folder.

If Chrome says there is no manifest, you selected the repository root instead of src.

Initial Setup

  1. Open the EdgeLang toolbar popup.
  2. Go to Settings.
  3. Select your native and target languages.
  4. Add one or more provider API keys.
  5. Click Validate Keys.
  6. Choose models if you want to override defaults.
  7. Run calibration.

Modes

  • Auto: chooses passive or active mode from the page language.
  • Passive: use this on target-language pages for comprehension practice.
  • Active: use this on native-language pages for recall and translation practice.

Toolbar and Popup

The popup gives you fast access to enable or disable the extension, switch modes, pause learning, manage the current site, open statistics, and launch settings.

While a page is being analyzed, the toolbar shows a flashing badge dot and the popup displays Processing....

If no cues appear, the popup also shows a blocker reason such as api_keys_not_configured, site_disabled, paused, or insufficient_text.

Visual Cue Settings

EdgeLang supports several cue styles and a configurable highlight color.

  • Underline
  • Background tint
  • Corner dot
  • Border
  • Custom highlight color, defaulting to light red

Language Settings

Both native and target selectors include a broad supported list including English, Russian, Hebrew, Spanish, French, German, Arabic, Chinese, Japanese, Korean, Hindi, Ukrainian, Thai, and more.

Provider Configuration

EdgeLang currently supports multiple AI backends and model selection from the options page.

  • OpenAI
  • Anthropic
  • Google Gemini
  • Groq
  • OpenRouter

The project uses a browser-safe routing layer inspired by ModelMesh so provider failover and selection stay manageable inside the extension.

Troubleshooting

  • Make sure at least one AI provider key is configured.
  • Check that the current site is not blocked by the blacklist or missing from the whitelist.
  • Confirm the extension is not paused.
  • Wait for dynamic sites to finish hydrating if you just opened a page like a news site.
  • Use the popup blocker reason if the page still shows no cues.
